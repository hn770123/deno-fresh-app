# ログイン機能のテスト実装計画

## テスト内容の検討結果

### E2Eテストシナリオ: ログインフロー
- **目的**: ユーザーが正しい資格情報を使用してログインし、トップページ（保護されたコンテンツ）にアクセスできることを確認する。
- **前提条件**:
  - データベースにテストユーザーが存在すること。
- **テスト手順**:
  1. `/login` ページにアクセスする。
  2. ユーザー名に `testuser` を入力する。
  3. パスワードに `password123` を入力する。
  4. ログインボタンをクリックする。
- **期待される結果**:
  1. トップページ `/` にリダイレクトされること。
  2. ページ内に「Hello World」という見出しが表示されていること。

## 実装計画

### 1. データベース初期化時のテストユーザー作成
- `db.ts` に、アプリケーション起動時にテストユーザーを自動生成するロジックを追加する。
- すでに `testuser` が存在する場合は作成をスキップする。
- セキュリティのため、パスワードは `auth_utils.ts` の `hashPassword` を使用してハッシュ化して保存する。

### 2. E2Eテストコードのリファクタリング
- `tests/e2e_test.ts` 内で行っている手動のユーザー作成処理を削除する。
- `db.ts` をインポートすることで、自動的にテストユーザーが用意される状態を利用する。
- Playwright を使用したブラウザ操作のコードを整理し、可読性を高める。

### 3. 動作確認
- `deno task build` により本番用アセットをビルドする。
- `deno task test` を実行し、テストが成功することを確認する。
